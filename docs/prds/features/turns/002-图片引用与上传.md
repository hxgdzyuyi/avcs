# 004 图片引用与上传

状态：Draft  
领域：`lib/avcs/turns`、`lib/avcs/assets`

## 1. 目标

用户可以把项目图片作为下一次 Agent 指令的引用，也可以在聊天区上传新图片。引用图片会随用户文本一起传给 Agent，用于基于现有图片继续迭代。

## 2. 图片引用来源

图片引用可以来自：

1. 画板 `Reference` 操作。
2. 点击聊天中的图片资产行。
3. 聊天区上传图片成功后自动加入。
4. 在 Composer 中粘贴剪贴板截图或图片，上传成功后自动加入。
5. Board 预览 dialog 底部描述编辑发送时自动携带当前预览图片。
6. Board 预览 dialog 中的临时 mask 编辑；发送时自动携带原图和 mask 图。
7. 后续导入或扫描得到的资产被用户手动选择。

## 3. 聊天区上传

上传流程适用于文件选择器上传和剪贴板图片粘贴：

1. 用户在 composer 中点击添加图片按钮。
2. 或者用户在 composer 中使用 `Ctrl+V` / `Cmd+V` 粘贴剪贴板图片。
3. 前端通过 Phoenix HTTP API 上传图片文件。
4. 后端将文件保存到当前项目 `work/` 目录。
5. 后端计算 hash 并执行项目内去重。
6. 后端创建或复用 asset 记录。
7. 上传成功的 asset 自动加入当前聊天输入引用列表。

上传和粘贴图片都属于工作素材，必须进入 `work/`，不能保存到 `output/`，也不自动创建画板对象。前端不得把剪贴板图片以 base64 写入本地状态或数据库。

Mask 上传属于预览 dialog 的控制引用，保存到项目内受控临时目录，asset 来源为 `mask`。Mask 不进入 `output/`，不自动创建画板对象，也不显示为普通 Work 素材。

## 4. 引用展示

Composer 中需要展示当前引用图片缩略图：

1. 缩略图显示在输入器上方或内部顶部。
2. 支持单个移除。
3. 长文件名需要省略，不撑破布局。
4. Agent 运行中仍可调整下一条草稿的引用。
5. 粘贴或上传过程中显示 loading 占位；失败时显示错误并允许用户移除失败占位。

## 5. Agent 入参

带图片引用发送消息后，后端需要向 Codex app-server 传入：

1. 用户文本。
2. 当前 thread ID。
3. 当前项目 `cwd`。
4. 聊天输入区当前引用的图片路径或引用信息。
5. 必要的图片元信息，例如 MIME 类型、尺寸或来源。
6. 如本轮为 mask 编辑，传入原图路径、mask 图路径和 `mask_edit` 元信息，并附加视觉参考指令。

React 不直接拼接本地路径读取图片，也不直接把任意本地路径传给 Agent。

## 6. API 与事件

HTTP API：

1. `assets:upload`：聊天区上传图片。
2. `assets:mask`：Board 预览 dialog 上传本轮 mask PNG。
3. 图片预览路由：通过受控 HTTP 路由返回项目资产文件。

WebSocket 事件：

1. `assets:reference`：更新当前聊天输入区引用的图片资产。
2. `asset:referenced`：图片资产被加入当前聊天输入引用。
3. `message:send`：发送用户文本和当前引用资产列表。

`message:send` 支持可选 `mask_edit`：

```json
{
  "asset_ids": ["base_asset_id", "mask_asset_id"],
  "mask_edit": {
    "mode": "visual_reference",
    "base_asset_id": "base_asset_id",
    "mask_asset_id": "mask_asset_id",
    "mask_semantics": "white_edit_black_keep"
  }
}
```

Board 预览 dialog 底部描述编辑发送消息时，前端使用 `message:send` 并传入 `{ text, asset_ids: [asset_id] }`；该图片引用只作用于本轮消息，不写入主 Composer 的引用列表。

如果用户涂抹了 mask，前端先通过 `assets:mask` 上传 mask，再使用 `message:send` 传入 `{ text, asset_ids: [base_asset_id, mask_asset_id], mask_edit }`。Mask 编辑是视觉参考式能力，不保证等同正式 mask API 的像素级保持。

## 7. 错误处理

需要覆盖：

1. 上传失败。
2. 格式不支持。
3. 文件超过大小限制。
4. 路径或权限问题。
5. 资产文件丢失或无法读取。
6. 引用资产不属于当前项目。
7. 粘贴图片仍在上传时发送消息。
8. Mask 上传失败、mask 不是 PNG、mask 不属于当前项目或 mask asset 来源不正确。

错误信息必须说明失败来源和可恢复动作。

## 8. 验收标准

1. 用户可以在聊天区上传图片。
2. 上传成功后图片保存到当前项目 `work/` 目录。
3. 上传成功后图片自动加入聊天输入引用。
4. 用户可以在 Composer 中粘贴剪贴板图片，粘贴成功后图片保存到 `work/` 并自动加入聊天输入引用。
5. 用户可以通过画板 `Reference` 操作加入聊天引用。
6. 用户可以在 Board 预览 dialog 底部发送描述，并自动携带当前预览图片作为本轮引用。
7. 用户可以移除单个引用。
8. 带图片引用发送消息后，Agent 能收到对应图片路径或引用。
9. 上传和引用过程不绕过 Phoenix API。
10. Composer 上传或粘贴的 `work/` 图片不会出现在画板中。
11. Board 预览 dialog 的 mask 编辑发送能携带原图和 mask 两个引用，并把 `mask_edit` 写入 user message payload。
