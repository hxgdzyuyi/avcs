# 004 图片引用与上传

状态：Draft  
领域：`lib/avcs/turns`、`lib/avcs/assets`

## 1. 目标

用户可以把项目图片作为下一次 Agent 指令的引用，也可以在聊天区上传新图片。引用图片会随用户文本一起传给 Agent，用于基于现有图片继续迭代。

## 2. 图片引用来源

图片引用可以来自：

1. 点击画板图片对象。
2. 点击聊天中的图片资产行。
3. 聊天区上传图片成功后自动加入。
4. 后续导入或扫描得到的资产被用户手动选择。

## 3. 聊天区上传

上传流程：

1. 用户在 composer 中点击添加图片按钮。
2. 前端通过 Phoenix HTTP API 上传图片文件。
3. 后端将文件保存到当前项目 `work/` 目录。
4. 后端计算 hash 并执行项目内去重。
5. 后端创建或复用 asset 记录。
6. 上传成功的 asset 自动加入当前聊天输入引用列表。

上传图片属于工作素材，必须进入 `work/`，不能保存到 `output/`。

## 4. 引用展示

Composer 中需要展示当前引用图片缩略图：

1. 缩略图显示在输入器上方或内部顶部。
2. 支持单个移除。
3. 长文件名需要省略，不撑破布局。
4. Agent 运行中仍可调整下一条草稿的引用。

## 5. Agent 入参

带图片引用发送消息后，后端需要向 Codex app-server 传入：

1. 用户文本。
2. 当前 thread ID。
3. 当前项目 `cwd`。
4. 聊天输入区当前引用的图片路径或引用信息。
5. 必要的图片元信息，例如 MIME 类型、尺寸或来源。

React 不直接拼接本地路径读取图片，也不直接把任意本地路径传给 Agent。

## 6. API 与事件

HTTP API：

1. `assets:upload`：聊天区上传图片。
2. 图片预览路由：通过受控 HTTP 路由返回项目资产文件。

WebSocket 事件：

1. `assets:reference`：更新当前聊天输入区引用的图片资产。
2. `asset:referenced`：图片资产被加入当前聊天输入引用。
3. `message:send`：发送用户文本和当前引用资产列表。

## 7. 错误处理

需要覆盖：

1. 上传失败。
2. 格式不支持。
3. 文件超过大小限制。
4. 路径或权限问题。
5. 资产文件丢失或无法读取。
6. 引用资产不属于当前项目。

错误信息必须说明失败来源和可恢复动作。

## 8. 验收标准

1. 用户可以在聊天区上传图片。
2. 上传成功后图片保存到当前项目 `work/` 目录。
3. 上传成功后图片自动加入聊天输入引用。
4. 用户可以从画板点击图片加入聊天引用。
5. 用户可以移除单个引用。
6. 带图片引用发送消息后，Agent 能收到对应图片路径或引用。
7. 上传和引用过程不绕过 Phoenix API。
