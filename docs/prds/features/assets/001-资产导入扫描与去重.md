# 005 资产导入扫描与去重

状态：Draft  
领域：`lib/avcs/assets`

## 1. 目标

管理项目中的图片资产，支持导入、上传、扫描和预览，并在项目内按 hash 去重。资产文件和资产记录必须归属于当前项目。

## 2. Asset 定义

Asset 表示项目中的素材文件，MVP 主要是图片。

Asset 需要记录：

1. 文件路径。
2. 文件类型和 MIME 类型。
3. 宽高、大小、hash 等基础元数据。
4. 来源：生成、导入、聊天区上传、用户手动放入项目文件夹、Agent 引用。
5. 关联的 thread、turn、item。
6. 生成提示词或加工指令。
7. 预览 dialog mask 编辑生成的控制引用，来源为 `mask`。

## 3. 目录归属

`work/` 用于保存：

1. 导入图片。
2. 聊天区上传图片，包括 Composer 剪贴板粘贴图片。
3. 用户手动放入的参考素材。
4. 需要继续加工的工作素材。

`output/` 用于保存：

1. Agent 生成图片。
2. 图片加工结果。
3. 后续导出的结果文件。
4. 用户从 Work 手动提升到 Output 的图片副本。

生成图片不能保存到 `work/`。

`.avcs/cache/temp/masks/` 用于保存 Board 预览 dialog 本轮 mask 编辑导出的 PNG。Mask asset 属于项目内受控文件，但不是普通素材，不进入 `output/`，不创建 board item，默认不显示在 Work 素材列表。

## 4. 导入与扫描

导入图片：

1. 接收用户指定的本地图片路径。
2. 将文件复制到当前项目 `work/`。
3. 计算 hash。
4. 写入或复用 asset 记录。
5. 创建必要的 asset link；不创建默认 board item。

扫描项目图片：

1. 默认扫描 `<project>/work/` 和 `<project>/output/` 内允许的图片格式。
2. 计算 hash。
3. 新增缺失的 asset 记录。
4. 只为 `output/` 下的图片创建或保留 board item；`work/` 下的图片仅作为素材资产和聊天引用。
5. 对缺失文件、重复文件和不支持格式给出明确结果。

扫描不需要主动扫描 `.avcs/cache/temp/masks/`；mask 文件由 `assets:mask` 创建并登记。

手动提升到 Output：

1. 用户可以从 Board 的 Work tab 把 Work asset 拖拽到 Output tab 或 Output 画板区域。
2. 用户也可以把本地图片文件直接拖到 Output 画板区域；前端通过 Phoenix multipart API 上传，不使用浏览器默认打开文件行为。
3. 前端提交 asset id 或浏览器 File，以及可选画板世界坐标，不直接复制本地文件路径。
4. 后端校验 asset 属于当前项目、文件存在、不是 mask asset。
5. 后端把图片复制到 `<project>/output/`，并将该 asset 的 canonical 路径更新为 `output/...`。
6. 在当前 `assets.hash` 项目内唯一约束下，不为同一 hash 新增第二条 asset 记录；旧 `work/` 文件可保留在磁盘上，但不再作为当前 Work asset 展示。
7. 提升后的 asset 创建或恢复 output board item，来源记录为 `output_copy`、`output_upload` 或等价枚举，避免误标为 Agent 生成图。

## 5. Hash 去重

同一项目内相同 hash 的图片不要重复创建资产文件。必要时只新增关联记录，例如 asset 与 thread、turn、item 或来源之间的关联。

项目 SQLite 中 `assets.hash` 应保持项目内唯一，或通过等效约束保证不会重复创建资产记录。

## 6. 预览与文件动作

读取图片预览必须通过受控 HTTP 路由返回项目资产文件，避免前端直接拼接任意本地路径。

打开所在文件夹或复制本地路径必须由后端校验 asset 属于当前项目后执行系统操作或返回规范化路径。

## 7. API 与事件

HTTP API：

1. `assets:import`：导入图片。
2. `assets:upload`：聊天区上传图片，也用于 Composer 剪贴板粘贴图片。
3. `assets:mask`：保存 Board 预览 dialog 导出的 mask PNG，创建 `source = "mask"` 的 asset。
4. `assets:copy_to_output`：把已有 Work asset 复制并提升到 Output。
5. `assets:upload_to_output`：把浏览器拖入 Output 的本地图片文件上传并保存到 Output。
6. `assets:scan`：扫描项目目录图片。
7. 图片预览路由。
8. 打开所在文件夹。
9. 复制本地路径。

WebSocket 事件：

1. `assets:list`：读取资产列表。
2. `asset:created`：新增图片资产。
3. `assets:updated`：资产列表更新。

## 8. 错误处理

需要覆盖：

1. 资产文件丢失或无法读取。
2. 上传或导入失败。
3. 图片格式不支持。
4. 路径校验失败。
5. 文件权限不足。
6. hash 计算失败。
7. 图片预览读取失败。
8. Mask 上传不是 PNG、尺寸无法读取或 base asset 不存在。
9. Work asset 提升到 Output 时文件缺失、不是普通素材或写入 `output/` 失败。

资产文件丢失时，画板对象显示缺失占位，并提供重新扫描或打开文件夹入口。

## 9. 验收标准

1. 导入、上传和扫描图片时，同一项目内相同 hash 的图片不会重复创建资产文件。
2. 资产元数据写入项目 SQLite。
3. 图片预览通过 Phoenix 受控 HTTP 路由返回。
4. 打开所在文件夹和复制路径需要后端校验 asset 属于当前项目。
5. 扫描 `work/` 和 `output/` 能补齐缺失 asset 记录。
6. 只有 `output/` 图片会进入画板；导入、上传和 Composer 粘贴保存在 `work/` 的图片不会显示在画板。
7. Mask asset 不会进入画板，也不会作为普通 Work 素材展示。
8. 用户把 Work asset 或本地图片文件拖拽到 Output 后，图片复制到 `output/`，asset canonical 路径变为 `output/...`，并创建或恢复一个 Output board item。
