# 005 资产导入扫描与去重

状态：Draft  
领域：`lib/avcs/assets`

## 1. 目标

管理项目中的图片资产，支持导入、上传、扫描和预览，并在项目内按 hash 去重。资产文件和资产记录必须归属于当前项目。

## 2. Asset 定义

Asset 表示项目中的素材文件，MVP 主要是图片。

Asset 需要记录：

1. 文件路径。
2. 文件类型和 MIME 类型。
3. 宽高、大小、hash 等基础元数据。
4. 来源：生成、导入、聊天区上传、用户手动放入项目文件夹、Agent 引用。
5. 关联的 thread、turn、item。
6. 生成提示词或加工指令。

## 3. 目录归属

`work/` 用于保存：

1. 导入图片。
2. 聊天区上传图片。
3. 用户手动放入的参考素材。
4. 需要继续加工的工作素材。

`output/` 用于保存：

1. Agent 生成图片。
2. 图片加工结果。
3. 后续导出的结果文件。

生成图片不能保存到 `work/`。

## 4. 导入与扫描

导入图片：

1. 接收用户指定的本地图片路径。
2. 将文件复制到当前项目 `work/`。
3. 计算 hash。
4. 写入或复用 asset 记录。
5. 创建必要的 asset link 和默认 board item。

扫描项目图片：

1. 默认扫描 `<project>/work/` 和 `<project>/output/` 内允许的图片格式。
2. 计算 hash。
3. 新增缺失的 asset 记录。
4. 对缺失文件、重复文件和不支持格式给出明确结果。

## 5. Hash 去重

同一项目内相同 hash 的图片不要重复创建资产文件。必要时只新增关联记录，例如 asset 与 thread、turn、item 或来源之间的关联。

项目 SQLite 中 `assets.hash` 应保持项目内唯一，或通过等效约束保证不会重复创建资产记录。

## 6. 预览与文件动作

读取图片预览必须通过受控 HTTP 路由返回项目资产文件，避免前端直接拼接任意本地路径。

打开所在文件夹或复制本地路径必须由后端校验 asset 属于当前项目后执行系统操作或返回规范化路径。

## 7. API 与事件

HTTP API：

1. `assets:import`：导入图片。
2. `assets:upload`：聊天区上传图片。
3. `assets:scan`：扫描项目目录图片。
4. 图片预览路由。
5. 打开所在文件夹。
6. 复制本地路径。

WebSocket 事件：

1. `assets:list`：读取资产列表。
2. `asset:created`：新增图片资产。
3. `assets:updated`：资产列表更新。

## 8. 错误处理

需要覆盖：

1. 资产文件丢失或无法读取。
2. 上传或导入失败。
3. 图片格式不支持。
4. 路径校验失败。
5. 文件权限不足。
6. hash 计算失败。
7. 图片预览读取失败。

资产文件丢失时，画板对象显示缺失占位，并提供重新扫描或打开文件夹入口。

## 9. 验收标准

1. 导入、上传和扫描图片时，同一项目内相同 hash 的图片不会重复创建资产文件。
2. 资产元数据写入项目 SQLite。
3. 图片预览通过 Phoenix 受控 HTTP 路由返回。
4. 打开所在文件夹和复制路径需要后端校验 asset 属于当前项目。
5. 扫描 `work/` 和 `output/` 能补齐缺失 asset 记录。
