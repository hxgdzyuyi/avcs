# 007 图片生成与结果入库

状态：Draft  
领域：`lib/avcs/assets`、`lib/avcs/agent`

## 1. 目标

用户通过自然语言请求 Agent 生成图片时，Avcs 必须真实接入 Codex built-in `image_gen`，捕获生成结果并保存到当前项目 `output/` 目录，同时写入 asset、turn/item 和 board item。

## 2. 用户场景

用户在当前 thread 中输入自然语言，例如：

```text
给一家咖啡店生成三张简洁风格的 logo 概念图
```

Avcs 将请求交给 Codex app-server。Agent 生成图片后，Avcs 将图片保存到项目 `output/` 目录，并在右侧画板展示。

## 3. 生成约束

1. 图片生成必须真实接入 Codex built-in `image_gen`。
2. MVP 不以 mock/stub 替代图片生成闭环。
3. Avcs 通过系统提示词和 turn 输入约束 Agent 在需要生成图片时优先调用 `image_gen`。
4. 生成结果必须回传为可被后端捕获的图片文件或图片引用。
5. 生成图片、加工结果和导出结果进入 `output/`，不能保存到 `work/`。

## 4. 结果处理流程

当 Agent 生成图片后，Avcs 需要完成：

1. 捕获图片文件或图片引用。
2. 将生成图片保存到当前项目 `output/` 目录。
3. 计算文件 hash，并在项目内按 hash 去重。
4. 写入或复用当前项目 SQLite 的 asset 记录。
5. 将 asset 与对应 thread、turn、item 关联。
6. 创建默认画板对象。
7. 通过 WebSocket 推送资产和画板更新。
8. 在右侧画板中自动刷新并展示。

## 5. 命名与来源

生成图片 asset 应记录：

1. 来源为 `generated` 或等价枚举。
2. 生成提示词或加工指令。
3. 关联 thread、turn、item。
4. 生成时间。
5. 文件 MIME 类型、尺寸、大小和 hash。

具体文件命名可以随实现调整，但必须稳定、可读，并避免覆盖同名文件。

## 6. 聊天与画板联动

图片生成成功后：

1. 聊天区显示图片资产行。
2. 资产行可点击定位并选中画板对象。
3. 画板自动展示生成图片对象。
4. 新生成资产默认放在当前视野中部，并按 turn 分组横向排列，避免堆叠在左上角。

## 7. 错误处理

需要覆盖：

1. `image_gen` 调用失败。
2. 生成结果无法捕获。
3. 图片保存到 `output/` 失败。
4. hash 计算失败。
5. asset 写库失败。
6. board item 创建失败。
7. 预览读取失败。

图片生成失败时，错误明确落在当前 turn 中，并保留用户输入。

## 8. 验收标准

1. Agent 通过真实 Codex built-in `image_gen` 生成图片。
2. 生成图片能保存到当前项目 `output/` 目录。
3. 生成图片写入或复用 asset 记录。
4. asset 与对应 thread、turn、item 建立关联。
5. 画板自动创建并展示图片对象。
6. 聊天区图片结果可定位到画板对象。
