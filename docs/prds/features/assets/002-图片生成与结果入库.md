# 007 图片生成与结果入库

状态：Draft  
领域：`lib/avcs/assets`、`lib/avcs/agent`

## 1. 目标

用户通过自然语言请求 Agent 生成图片时，Avcs 必须真实接入 Codex built-in `image_gen`，捕获生成结果并保存到当前项目 `output/` 目录，同时写入 asset、turn/item 和 board item。

## 2. 用户场景

用户在当前 thread 中输入自然语言，例如：

```text
给一家咖啡店生成三张简洁风格的 logo 概念图
```

Avcs 将请求交给 Codex app-server。Agent 生成图片后，Avcs 将图片保存到项目 `output/` 目录，并在右侧画板展示。

## 3. 生成约束

1. 图片生成必须真实接入 Codex built-in `image_gen`。
2. MVP 不以 mock/stub 替代图片生成闭环。
3. Avcs 通过系统提示词和 turn 输入约束 Agent 在需要生成图片时优先调用 `image_gen`。
4. 生成结果必须回传为可被后端捕获的图片文件或图片引用。
5. 生成图片、加工结果和导出结果进入 `output/`，不能保存到 `work/`。
6. Board 预览 dialog 的 mask 编辑结果仍按图片加工结果处理，进入 `output/`；mask PNG 本身只是控制引用，不作为生成结果。
7. Codex built-in `image_gen` 当前未暴露正式 mask 参数，mask 编辑采用“原图 + mask 图 + 文本指令”的视觉参考式调用。
8. 如果本轮携带 `data_provider`，Agent runner 必须先按 provider skill 获取来源数据，再进入 `avcs-imagegen` 图片生成流程；未携带 provider 时保持现有链路不变。

Composer 调色盘可以为本轮 prompt 附加图片生成偏好：

1. 支持比例：`auto`、`1:1`、`16:9`、`9:16`、`4:3`、`3:4`。
2. 支持数量：`1` 到 `4`。
3. 支持透明背景开关。
4. 这些参数由前端在发送前拼接到本轮 prompt 后面，例如 `Image settings: aspect ratio 16:9; image count 2; transparent background.`。
5. 调色盘参数只影响本次发送，不作为独立 API 字段或持久设置。

## 4. 结果处理流程

当 Agent 生成图片后，Avcs 需要完成：

1. 捕获图片文件或图片引用。
2. 将生成图片保存到当前项目 `output/` 目录。
3. 计算文件 hash，并在项目内按 hash 去重。
4. 写入或复用当前项目 SQLite 的 asset 记录。
5. 将 asset 与对应 thread、turn、item 关联。
6. 创建默认画板对象。
7. 通过 WebSocket 推送资产和画板更新。
8. 在右侧画板中自动刷新并展示。
9. 如果本轮使用 Data Provider，assistant 或图片结果 item 的 payload 应保留 `provider_context`，至少包含 provider 元信息，并尽量保留 provider 返回的标题、日期、来源、图片路径或失败原因摘要。

如果本轮是 mask 编辑，asset 的 prompt 或关联 item payload 应能追溯到原始描述和 `mask_edit` 元信息。

## 5. 命名与来源

生成图片 asset 应记录：

1. 来源为 `generated` 或等价枚举。
2. 生成提示词或加工指令。
3. 关联 thread、turn、item。
4. 生成时间。
5. 文件 MIME 类型、尺寸、大小和 hash。

具体文件命名可以随实现调整，但必须稳定、可读，并避免覆盖同名文件。

## 6. 聊天与画板联动

图片生成成功后：

1. 聊天区显示图片资产行。
2. 资产行可点击定位并选中画板对象。
3. 画板自动展示生成图片对象。
4. 新生成资产默认放在当前视野中部，并按 turn 分组横向排列，避免堆叠在左上角。

## 7. 错误处理

需要覆盖：

1. `image_gen` 调用失败。
2. 生成结果无法捕获。
3. 图片保存到 `output/` 失败。
4. hash 计算失败。
5. asset 写库失败。
6. board item 创建失败。
7. 预览读取失败。
8. Mask 编辑结果偏离遮罩区域预期时，需在用户可见输出中避免承诺正式 mask API 的精确保持能力。
9. Data Provider 加载失败、provider slug 不受支持、provider 执行失败或返回非图片条目时，需要明确落入当前 turn 的可追溯错误或 assistant 说明中。

图片生成失败时，错误明确落在当前 turn 中，并保留用户输入。

## 8. 验收标准

1. Agent 通过真实 Codex built-in `image_gen` 生成图片。
2. 生成图片能保存到当前项目 `output/` 目录。
3. 生成图片写入或复用 asset 记录。
4. asset 与对应 thread、turn、item 建立关联。
5. 画板自动创建并展示图片对象。
6. 聊天区图片结果可定位到画板对象。
7. 基于 mask 的图片加工结果按生成结果入库到 `output/`，mask 控制图不作为输出图展示。
8. 使用 `avcs-data-prodiver-apod` 时，turn 配置和输出 item payload 能回溯 provider 选择与 provider 返回摘要。
