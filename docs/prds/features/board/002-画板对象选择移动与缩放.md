# 009 画板对象选择移动与缩放

状态：Draft  
领域：`lib/avcs/board`

## 1. 目标

用户可以选中画板图片对象，看到选中框、控制点、图片名称和显示尺寸，并能移动或调整显示尺寸。位置和尺寸保存到项目 SQLite，重新打开项目后恢复。

## 2. 选中态

图片对象需要满足：

1. 图片本体使用 `img`，`object-fit: contain`。
2. 透明 PNG 使用棋盘或浅色背景提示。
3. 未选中时只有极轻边框或无边框。
4. 选中时显示蓝色 `2px` 选中框、四角控制点、文件名和显示尺寸。
5. 控制点、名称、尺寸标注和操作按钮都用 DOM 元素实现。
6. 多选时显示 `n selected`，但不遮挡图片主体。
7. 在 camera zoom 下，图片本体随视图缩放，选中框、控制点、文件名和尺寸标注保持固定屏幕尺寸。
8. 多选时每个对象显示轻量选中框，并额外显示覆盖全部对象的聚合选区边界。
9. 默认工具为 `Select`；右侧浮动工具栏提供 `Select` 与 `Pan` 互斥切换。

选中态示例：

```text
┌──────────────────────┐
│        image         │
└──────────────────────┘
coffee-logo-v03.png
896 x 1200
```

## 3. 移动

移动流程：

1. 用户在 Select 模式下拖动画板对象。
2. 前端把屏幕拖拽 delta 除以当前 `camera.zoom`，转换成世界坐标位移。
3. 如果对象属于当前选区，拖动会移动整个选区；如果对象未选中，先切换为单选再移动。
4. 前端即时更新对象位置。
5. pointer up 后通过 WebSocket 保存世界坐标 `x`、`y`。
6. 多选移动使用 `board:items:update` 批量保存。
7. 保存成功后更新项目 SQLite。
8. 保存失败时回滚到保存前位置、重新拉取画板对象，或展示可重试提示。

区域选择：

1. Select 模式下空白处点击清空选区。
2. Select 模式下空白处拖拽绘制区域框选，命中规则为矩形相交。
3. 按住 Shift/Cmd/Ctrl 框选时追加到当前选区；普通框选替换当前选区。
4. Pan 模式下空白拖拽只移动 camera，不选择、不移动对象。

## 4. 缩放

缩放流程：

1. 用户拖动控制点调整图片对象显示尺寸。
2. 调整只影响显示宽高，不修改原始图片文件。
3. 前端把屏幕拖拽 delta 除以当前 `camera.zoom`，转换成世界尺寸变化。
4. 前端即时更新 `display_width` 和 `display_height`。
5. pointer up 后通过 WebSocket 保存世界尺寸。
6. 保存失败时回滚、重新拉取画板对象，或展示可重试提示。

MVP 不要求裁切、旋转、蒙版或像素编辑。

## 5. 引用与操作

画板对象操作：

1. 普通点击 Output 图片对象打开全屏图片预览 dialog。
2. 拖动画板对象时只移动对象，不打开预览。
3. Shift/Cmd/Ctrl 追加选择时只更新选区，不打开预览。
4. `Reference`：将选中图片加入聊天输入引用。
5. 预览 dialog 底部描述编辑发送时，消息自动携带当前预览图片作为本轮唯一图片引用。
6. `Reveal`：打开所在文件夹。
7. `Copy path`：复制本地路径。
8. `Fit selected`：移动 camera，使当前选中对象进入 viewport 中央。
9. `Fit all`：移动并缩放 camera，使当前已加载且可见的画板对象进入 viewport。
10. 多选时显示顶部浮动布局工具条，包含 Align 和 Tidy 两组操作。
11. Align 支持左/水平居中/右/上/垂直居中/下对齐。
12. Align 菜单内同时提供 `Normalize Width` 和 `Normalize Height`，按 primary selection 统一宽度或高度。
13. Tidy 支持横向或纵向整理间距，多选时可用；按对象中心点排序后，将相邻对象整理为 16px 间距。

打开所在文件夹和复制路径必须通过 Phoenix API 进行后端校验。

## 6. WebSocket 事件

客户端发送：

1. `assets:select`：更新画板当前选中的图片资产。
2. `board:item:move`：更新画板对象位置。
3. `board:item:resize`：更新画板对象显示尺寸。
4. `board:items:update`：批量更新多个 output board item 的位置和显示尺寸。
5. `assets:reference`：将资产加入聊天输入引用。

服务端推送：

1. `board:item:updated`：画板对象位置、尺寸或层级更新。
2. `asset:referenced`：图片资产被加入当前聊天输入引用。

## 7. 错误处理

需要覆盖：

1. 画板对象位置保存失败。
2. 画板对象尺寸保存失败。
3. 资产文件丢失。
4. 打开所在文件夹失败。
5. 复制路径失败。
6. 多端或多窗口状态冲突。

拖动或缩放保存失败时，不应让前端和数据库长期不一致；前端可以通过重新拉取 `board:items:list` 恢复服务端状态。

## 8. 验收标准

1. 用户可以选中画板图片对象。
2. 选中时看到选中框、控制点、图片名称和显示尺寸。
3. 用户可以移动画板图片对象。
4. 用户可以调整画板图片对象显示尺寸。
5. 位置和显示尺寸能保存并在再次打开项目后恢复。
6. 用户点击 Output 画板图片后会打开全屏图片预览 dialog。
7. 拖动画板对象和 Shift/Cmd/Ctrl 追加选择不会打开预览 dialog。
8. 用户通过 `Reference` 或预览 dialog 底部发送把图片作为引用传给 Agent。
9. 拖动、缩放时不发生文字覆盖图片主体或按钮重叠。
10. 在不同 zoom 下移动和 resize 仍保存世界坐标和世界尺寸，选中控件保持固定屏幕尺寸。
11. 默认空白拖拽是区域框选，切换 Pan 后空白拖拽才平移画板。
12. 多选移动、对齐、统一宽高和间距整理会保存到项目 SQLite。
