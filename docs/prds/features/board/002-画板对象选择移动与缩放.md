# 009 画板对象选择移动与缩放

状态：Draft  
领域：`lib/avcs/board`

## 1. 目标

用户可以选中画板图片对象，看到选中框、控制点、图片名称和显示尺寸，并能移动或调整显示尺寸。位置和尺寸保存到项目 SQLite，重新打开项目后恢复。

## 2. 选中态

图片对象需要满足：

1. 图片本体使用 `img`，`object-fit: contain`。
2. 透明 PNG 使用棋盘或浅色背景提示。
3. 未选中时只有极轻边框或无边框。
4. 选中时显示蓝色 `2px` 选中框、四角控制点、文件名和显示尺寸。
5. 控制点、名称、尺寸标注和操作按钮都用 DOM 元素实现。
6. 多选时显示 `n selected`，但不遮挡图片主体。
7. 在 camera zoom 下，图片本体随视图缩放，选中框、控制点、文件名和尺寸标注保持固定屏幕尺寸。
8. 多选时每个对象显示轻量选中框，并额外显示覆盖全部对象的聚合选区边界。
9. 默认工具为 `Select`；右侧浮动工具栏提供 `Select` 与 `Pan` 互斥切换。

选中态示例：

```text
┌──────────────────────┐
│        image         │
└──────────────────────┘
coffee-logo-v03.png
896 x 1200
```

## 3. 移动

移动流程：

1. 用户在 Select 模式下拖动画板对象。
2. 前端把屏幕拖拽 delta 除以当前 `camera.zoom`，转换成世界坐标位移。
3. 如果对象属于当前选区，拖动会移动整个选区；如果对象未选中，先切换为单选再移动。
4. 前端即时更新对象位置。
5. pointer up 后通过 WebSocket 保存世界坐标 `x`、`y`。
6. 多选移动使用 `board:items:update` 批量保存。
7. 保存成功后更新项目 SQLite。
8. 保存失败时回滚到保存前位置、重新拉取画板对象，或展示可重试提示。

区域选择：

1. Select 模式下空白处点击清空选区。
2. Select 模式下空白处拖拽绘制区域框选，命中规则为矩形相交。
3. 按住 Shift/Cmd/Ctrl 框选时追加到当前选区；普通框选替换当前选区。
4. Pan 模式下空白拖拽只移动 camera，不选择、不移动对象。

## 4. 缩放

缩放流程：

1. 用户拖动控制点调整图片对象显示尺寸。
2. 调整只影响显示宽高，不修改原始图片文件。
3. 前端把屏幕拖拽 delta 除以当前 `camera.zoom`，转换成世界尺寸变化。
4. 前端即时更新 `display_width` 和 `display_height`。
5. pointer up 后通过 WebSocket 保存世界尺寸。
6. 保存失败时回滚、重新拉取画板对象，或展示可重试提示。

MVP 不要求裁切、旋转、正式蒙版图层或像素编辑。Board 预览 dialog 可提供临时 mask 涂抹并导出 OpenAI edits 兼容的 alpha PNG mask：涂抹区域表示需要编辑，文件中对应区域透明，未涂抹区域不透明。该操作不修改原始图片文件。

## 5. 引用与操作

画板对象操作：

1. 普通点击 Output 图片对象只选中对象，不打开预览 dialog。
2. 拖动画板对象时只移动对象，不打开预览。
3. Shift/Cmd/Ctrl 追加选择时只更新选区，不打开预览。
4. 单选时显示 `board-layout-toolbar`，其中“编辑”按钮用于打开全屏图片预览 dialog。
5. `board-layout-toolbar` 的 `Reference` 按钮将当前选中图片加入聊天输入引用。
6. 预览 dialog 底部描述编辑发送时，消息自动携带当前预览图片作为本轮唯一图片引用；如用户涂抹 mask，则先上传 mask，再携带原图与 mask 两个引用。
7. `Reveal`：打开所在文件夹。
8. `Copy path`：复制本地路径。
9. `Fit selected`：移动 camera，使当前选中对象进入 viewport 中央。
10. `Fit all`：移动并缩放 camera，使当前已加载且可见的画板对象进入 viewport。
11. 多选时显示顶部浮动布局工具条，包含 Align 和 Tidy 两组操作。
12. Align 支持左/水平居中/右/上/垂直居中/下对齐。
13. Align 菜单内同时提供 `Normalize Width` 和 `Normalize Height`，按 primary selection 统一宽度或高度。
14. Tidy 支持横向或纵向整理间距，多选时可用；按对象中心点排序后，将相邻对象整理为 16px 间距。
15. 右侧浮动工具栏提供 `Arrange all by actual scale`：第一次点击只弹出确认；用户点击“确定”后，按同一个真实尺寸 scale 重新计算当前可见 Output 画板对象的 `display_width` / `display_height`，再按行优先布局重排并通过 `board:items:update` 保存。
16. 右键 Output 画板对象时显示层级菜单，包含上移一层、下移一层、移动至顶层、移动至底层；多选时对当前选区整体生效并保持选区内部相对顺序。
17. `Delete` / `Backspace` 删除单个选中的 Output 图片，删除前继续使用 ConfirmDialog 二次确认；多选时不执行批量删除，只提示一次只能删除一个文件。
18. 图层层级快捷键复用 `board:items:update`：`]` 上移一层、`[` 下移一层、`Shift+]` 移动至顶层、`Shift+[` 移动至底层；多选时对当前选区整体生效并保持选区内部相对顺序。
19. 焦点在输入框、textarea、CodeMirror、select、contenteditable、IME 组合输入中，或已有 modal 打开时，画板和图层全局快捷键不生效。
20. 暂不实现 board item 复制、剪切、粘贴或复制图层实例快捷键。
21. 右侧浮动工具栏提供 Undo / Redo，撤销或重做当前会话内的移动、resize、Actual size、Align、Normalize、Tidy、Arrange 和层级调整。
22. Undo / Redo 快捷键：`Cmd/Ctrl+Z` 撤销，`Cmd/Ctrl+Shift+Z` 重做，Windows/Linux 额外支持 `Ctrl+Y` 重做。
23. Undo / Redo 只记录 `x`、`y`、`display_width`、`display_height`、`z_index`，不恢复删除文件、复制图层实例、引用、camera 或预览 dialog 状态。
24. Work tab 的图片或本地图片文件可以拖到 Output tab 或 Output 画板区域；成功后创建或恢复 Output board item，并选中、聚焦该对象。
25. Work 图片或本地图片文件拖到 Output 画板区域时，drop 点换算为世界坐标保存为初始 `x`、`y`；拖到 Output tab 时使用默认排布。
26. Work 图片行保留一个复制到 Output 的按钮作为拖拽失败或不便拖拽时的 fallback。

打开所在文件夹和复制路径必须通过 Phoenix API 进行后端校验。

## 6. WebSocket 事件

客户端发送：

1. `assets:select`：更新画板当前选中的图片资产。
2. `board:item:move`：更新画板对象位置。
3. `board:item:resize`：更新画板对象显示尺寸。
4. `board:items:update`：批量更新多个 output board item 的位置、显示尺寸和层级；Undo / Redo 也复用该事件提交快照。
5. `assets:reference`：将资产加入聊天输入引用。

服务端推送：

1. `board:item:updated`：画板对象位置、尺寸或层级更新。
2. `asset:referenced`：图片资产被加入当前聊天输入引用。

## 7. 错误处理

需要覆盖：

1. 画板对象位置保存失败。
2. 画板对象尺寸保存失败。
3. 画板对象层级保存失败。
4. 资产文件丢失。
5. 打开所在文件夹失败。
6. 复制路径失败。
7. 多端或多窗口状态冲突。
8. Work 图片或本地图片文件复制到 Output 失败、文件缺失、格式不支持或 asset 不可提升。

拖动、缩放、布局、层级调整或 Undo / Redo 保存失败时，不应让前端和数据库长期不一致；前端可以通过重新拉取 `board:items:list` 恢复服务端状态。收到全量 `board:items` 或切换项目后，应清空当前会话内的 Undo / Redo 历史栈。

## 8. 验收标准

1. 用户可以选中画板图片对象。
2. 选中时看到选中框、控制点、图片名称和显示尺寸。
3. 用户可以移动画板图片对象。
4. 用户可以调整画板图片对象显示尺寸。
5. 位置和显示尺寸能保存并在再次打开项目后恢复。
6. 用户点击 Output 画板图片后只选中对象，不打开全屏图片预览 dialog。
7. 用户选中单张图片后，可以通过 `board-layout-toolbar` 的“编辑”按钮打开全屏图片预览 dialog。
8. 拖动画板对象和 Shift/Cmd/Ctrl 追加选择不会打开预览 dialog。
9. 用户通过 `Reference` 或预览 dialog 底部发送把图片作为引用传给 Agent。
10. 用户在预览 dialog 中涂抹 mask 后发送，Agent 收到原图、mask 图和视觉参考式 `mask_edit` 指令。
11. 拖动、缩放时不发生文字覆盖图片主体或按钮重叠。
12. 在不同 zoom 下移动和 resize 仍保存世界坐标和世界尺寸，选中控件保持固定屏幕尺寸。
13. 默认空白拖拽是区域框选，切换 Pan 后空白拖拽才平移画板。
14. 多选移动、对齐、统一宽高和间距整理会保存到项目 SQLite。
15. 用户可以通过右键菜单调整单个或多个 Output 画板对象的层级，刷新后层级保持一致。
16. 用户点击 `Arrange all by actual scale` 后需要再次确认；确认后所有当前可见 Output 画板对象按真实尺寸比例重排并保存，取消时不改变位置或显示尺寸。
17. 用户可以通过 `Delete` / `Backspace` 删除单个选中 Output 图片，并通过 `]`、`[`、`Shift+]`、`Shift+[` 调整层级，刷新后层级保持一致。
18. 用户可以通过 Undo / Redo 按钮或快捷键撤销、重做移动、缩放、布局整理和层级调整；刷新页面或切换项目后历史栈清空。
19. 用户把 Work 图片或本地图片文件拖到 Output，或点击复制到 Output 后，该图片在 Output 中被选中并聚焦，失败时显示明确错误且前端状态回到服务端状态。
