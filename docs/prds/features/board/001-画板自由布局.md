# 008 画板自由布局

状态：Draft  
领域：`lib/avcs/board`

## 1. 目标

画板是 Avcs 的主视觉区域，用白色自由工作区展示项目 `output/` 目录中的图片对象，支持图片摆放、预览、引用和轻量组织。MVP 使用普通 HTML DOM 元素实现，不使用 Canvas/WebGL/SVG 作为主渲染层。

## 2. 画板定位

画板类似轻量设计工具工作区，但 MVP 只做 DOM 元素级组织：

1. 白色自由工作区中展示 `output/` 图片对象。
2. 图片对象、选中边框、控制点、名称和尺寸标注都使用 DOM 元素。
3. 点击 Output 画板图片只选中对象；选中后通过 `board-layout-toolbar` 的“编辑”按钮打开全屏图片预览，加入引用通过 `Reference` 操作或预览 dialog 底部发送完成。
4. Work tab 展示 `work/` 图片素材；用户可把 Work 图片或本地图片文件拖到 Output tab / Output 画板区域，后端复制或上传到 `output/` 并创建画板对象。
5. 画板支持前端临时 viewport/camera 平移和缩放；缩放只影响视图，不改变对象保存的真实坐标和显示尺寸。
6. 不实现专业图层树、裁切、旋转、正式蒙版图层、文本对象、对齐吸附、小地图或复杂快捷键；仅允许图片预览 dialog 中的临时 mask 涂抹作为 Agent 视觉参考。

## 3. 画板对象数据

画板对象表示一个 `output/` asset 在右侧画板中的可视摆放实例。MVP 中一个 `output/` asset 默认对应一个画板对象；`work/` 中的上传、导入和 Composer 粘贴素材不自动创建画板对象，只有用户手动提升到 Output 后才创建。

画板对象需要记录：

1. 关联 asset。
2. 画板坐标，例如 `x`、`y`。
3. 显示宽高，例如 `display_width`、`display_height`。
4. 层级顺序，例如 `z_index`。
5. 所属 thread、turn 或 item 的来源标记，用于筛选和定位。
6. 创建时间和更新时间。
7. 历史编辑路径状态：`invalidated_at` 和 `invalidated_by_item_id`，用于隐藏旧路径生成的 Output 对象。

层级约定：`z_index` 越大越靠上；层级调整后应保存为稳定的正整数顺序。

历史编辑重跑时，与失效 turn/item 关联的 board item 标记为失效。默认 `Board.list_items/1` 和画板 UI 只返回当前有效路径的对象；对应图片文件和 `assets` 行保留，不从 `output/` 删除。

## 4. 画板结构

1. `BoardPane` 背景使用极浅灰，模拟大面积工作区。
2. `board-shell` 是固定 viewport，使用 `overflow: hidden` 接收 pan/zoom 事件。
3. `board-world` 是 DOM 内容层，通过 CSS transform 把世界坐标映射到屏幕坐标。
4. `board-overlay` 是屏幕层，不应用 camera transform，用于固定尺寸的选中框、控制点、名称和尺寸标注。
5. 画板对象使用 DOM 绝对定位元素，保存 `x`、`y`、`display_width`、`display_height`、`z_index`，并用 CSS `z-index` 体现前后层级。
6. 新生成或扫描到的 `output/` 资产默认放在当前视野中部，并按 turn 分组横向排列，避免堆叠在左上角。
7. 分组标题使用小字号，例如 `Current thread`、`Generated`、`Imported` 或 `Ungrouped`，不影响自由拖拽。
8. 用户可以触发按真实尺寸比例重排当前可见 Output 画板对象；该操作只更新对象位置和显示尺寸，不改变 camera 的持久状态和对象层级。
9. 用户可以在当前浏览器会话内撤销或重做 Output 画板对象的位置、显示尺寸和层级变更；历史栈不跨刷新、跨项目或跨窗口保存。
10. Work 图片或本地图片文件拖到 Output 画板区域时，drop 点按当前 camera 换算为世界坐标，作为新 board item 的初始位置；拖到 Output tab 时由后端使用默认排布。

视图状态：

1. `camera.x`、`camera.y` 表示 viewport 左上角对应的世界坐标。
2. `camera.zoom` 表示缩放倍率，默认 `1`，范围先限制为 `0.1` 到 `6`。
3. `camera` 是前端临时状态，不保存到项目 SQLite。
4. 切换项目或重新加载页面可以重置 camera，避免把个人视角写入项目业务数据。
5. `Cmd/Ctrl` + 滚轮缩放画板，普通滚轮只做平移或浏览，不强制缩放。
6. `Shift+2` 适配当前选中对象；`Cmd/Ctrl+0` 适配全部可见画板对象。
7. `Space` 按住期间临时进入平移状态，松开后恢复原工具模式；`H` 切换为 Pan 工具，`V` 切换为 Select 工具。
8. `Cmd/Ctrl+\` 显示或隐藏左栏与聊天栏，只影响前端工作台布局状态。

## 5. 筛选与高亮

画板支持按当前 thread、来源、时间筛选或高亮画板对象。

筛选规则：

1. 筛选改变只影响可见对象。
2. 不删除 board item。
3. 不改变对象坐标。
4. 筛选值使用业务语言，不直接暴露数据库字段，例如 `Generated` 优于 `generated`。
5. 筛选只作用于当前有效路径；已被历史编辑失效的 board item 不参与当前画板筛选和高亮。

## 6. 不做事项

MVP 不要求实现：

1. 小地图。
2. 对齐吸附。
3. 旋转。
4. 裁切。
5. 正式蒙版图层。
6. 图层面板。
7. 文本对象。
8. 命令面板、可自定义快捷键、系统级全局热键或复杂快捷键体系。
9. 持久化历史记录、历史面板、删除恢复或跨窗口 Undo / Redo 合并。
10. Canvas/WebGL/SVG 主画板。

平移和缩放必须使用 DOM/CSS transform 实现，不引入 Canvas/WebGL/SVG 主渲染层，并且缩放只影响视图，不改变 board item 保存的真实坐标和显示尺寸。图片预览 dialog 可使用局部 raster drawing layer 导出本轮 mask，但不得替代画板主渲染层。

## 7. WebSocket 事件

客户端发送：

1. `board:items:list`：读取画板对象列表。

服务端推送：

1. `board:item:created`：新增画板对象。
2. `board:item:updated`：画板对象位置、尺寸或层级更新。
3. `board:items`：历史编辑重跑或批量变更后刷新当前有效画板对象列表。

## 8. 验收标准

1. 右侧画板可以在白色自由工作区中自动展示项目 `output/` 图片对象。
2. 画板对象使用普通 HTML DOM 元素自由布局。
3. 画板不使用 Canvas/WebGL/SVG 作为主渲染层。
4. 新生成或扫描到的 `output/` 资产默认出现在当前视野中，不能全部堆叠在左上角。
5. 筛选或高亮不删除对象，也不改变坐标。
6. 用户可以平移和缩放画板视图，保存到数据库的对象坐标和显示尺寸仍保持世界坐标。
7. 画板对象层级在刷新或重新打开项目后保持一致。
8. 用户确认后，可以把当前可见 Output 画板对象按同一真实尺寸 scale 重排，刷新后位置和显示尺寸保持一致。
9. `V`、`H`、按住 `Space` 拖拽、`Cmd/Ctrl` + 滚轮、`Shift+2`、`Cmd/Ctrl+0` 和 `Cmd/Ctrl+\` 与对应工具按钮行为一致。
10. 当前会话内，用户可以撤销和重做画板对象移动、缩放、布局整理和层级调整；刷新页面后历史栈清空。
11. 历史消息编辑重跑后，旧路径生成的 Output board item 不再显示在当前画板，但对应输出文件仍保留在项目 `output/`。
12. 用户可以把 Work 图片或本地图片文件拖到 Output；成功后该图片进入 `output/`，在 Output 画板中选中并聚焦显示。
