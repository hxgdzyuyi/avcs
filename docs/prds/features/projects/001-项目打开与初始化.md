# 001 项目打开与初始化

状态：Draft  
领域：`lib/avcs/projects`

## 1. 目标

让用户以本地文件夹为项目单位进入 Avcs，并在项目文件夹中初始化项目数据库、素材目录和输出目录。全局 SQLite 保存软件级元数据和项目索引，项目业务数据全部保存在项目文件夹内。

## 2. 用户流程

1. 用户在左栏项目区点击文件夹加号入口。
2. 用户选择 `新建空白项目` 或 `使用现有文件夹`。空白项目由用户输入项目名，后端自动在全局软件设置 `projects.default_root` 指定的目录下创建项目目录，默认是 `~/Documents/Avcs`；使用现有文件夹时由用户输入本地绝对路径。
3. 前端调用 Phoenix 本地文件 API，不直接访问本地文件系统。
4. 后端规范化路径，创建或读取全局项目记录。
5. 后端在项目文件夹内初始化 `.avcs/project.sqlite3`、`work/`、`output/` 和缓存目录。
6. 打开成功后，项目出现在左栏项目列表中；前端进入三栏工作台，并恢复 thread、聊天记录、资产和画板对象。

首次启动时，如果 WebSocket join 后全局项目索引为空且没有当前项目，前端自动弹出 `新建空白项目` Dialog。用户取消后保留空状态，本次页面生命周期内不反复弹出；用户仍可通过左栏文件夹加号手动新建项目或打开现有文件夹。

## 3. 存储规则

全局索引库位于：

```text
~/.avcs/avcs.sqlite3
```

全局库保存软件级元数据和项目关联信息，不保存项目业务数据。项目关联信息包括：

1. 项目 ID。
2. 项目名称。
3. 项目文件夹路径。
4. 项目 SQLite 路径。
5. 左侧栏显示顺序 `sidebar_order`。
6. 归档时间、创建时间、更新时间、最近打开时间。

空白项目默认根目录来自全局软件设置 `projects.default_root`：

```text
~/Documents/Avcs
```

创建空白项目时，后端根据用户输入的项目名创建 `<projects.default_root>/<项目名>`。如果同名目录已存在，自动递增为 `<项目名> 2`、`<项目名> 3`，直到找到可用目录。`projects.default_root` 为空或不可用时，回退到 `~/Documents/Avcs`。

项目目录结构：

```text
<project>/
  .avcs/
    project.sqlite3
    cache/
      thumbnails/
      temp/
  work/
  output/
```

项目 SQLite 保存该项目自己的 thread、turn、item、asset、board item 和项目设置。

## 4. 交互与状态

未打开项目时：

1. 左栏展示项目列表和文件夹加号入口。
2. 文件夹加号菜单包含可点击的 `新建空白项目` 和 `使用现有文件夹`。
3. 当项目列表为空且没有通过 `project_path` 打开外部项目时，前端自动弹出一次 `新建空白项目` Dialog。
4. 聊天和画板禁用或展示空态。
5. 禁用 thread 新建和聊天发送。
6. 不展示独立的最近项目启动页；左栏项目列表是全局项目索引的主要入口。

再次打开同一文件夹时，Avcs 可以通过全局 SQLite 识别已有项目记录，并读取项目 SQLite 恢复历史数据。

如果 `projects.folder_path` 或 `projects.project_db_path` 指向的路径不存在，界面应展示明确错误，并要求用户重新打开正确的项目文件夹。MVP 不自动修复已移动项目文件夹的全局 SQLite 映射。

## 5. API 与事件

项目打开和创建由 Phoenix HTTP API 承担，逻辑上对应 `project:open_folder` 或 `project:create_blank`；其中 `project:create_blank` 默认创建到全局软件设置 `projects.default_root`，初始默认值为 `~/Documents/Avcs`。React 不直接读写文件系统，也不使用 Chromium File System Access API。

HTTP API：

1. `POST /api/project/create_blank`：创建空白项目。请求体为 `{ "name": "Project Name" }`。后端在全局软件设置 `projects.default_root` 下创建递增不重名的项目目录，并初始化项目。
2. `POST /api/project/open`：使用现有文件夹。请求体为 `{ "path": "/absolute/project/folder" }`。后端打开或初始化该绝对路径对应的项目。
3. `GET /api/project/sqlite_info`：读取当前项目 `.avcs/project.sqlite3` 的文件状态、SQLite PRAGMA 信息、关键表行数和最近优化时间。
4. `POST /api/project/sqlite_maintenance`：对当前项目 SQLite 执行维护。请求体为 `{ "action": "fast_optimize" | "deep_vacuum" }`；快速优化同步执行 `wal_checkpoint(TRUNCATE)` 与 `PRAGMA optimize`，深度整理通过后台任务执行 `VACUUM`。

WebSocket 相关事件：

1. `projects:list`：读取全局项目索引列表。
2. `project:current`：读取当前项目。
3. `project:select`：切换当前项目；只改变当前上下文，不更新最近打开时间，也不触发项目列表重排。
4. `project:reorder`：按客户端传入的完整项目 ID 列表重写未归档项目的 `sidebar_order`。
5. `project:archive`：归档项目索引项，隐藏在默认项目列表中，不删除本地项目文件夹。
6. `project:delete`：删除全局 SQLite 中的项目引用，不删除本地项目文件夹或项目 SQLite。
7. `project:updated`：服务端推送当前项目状态变化。
8. `projects:updated`：服务端推送项目列表变化。
9. `project:sqlite:maintenance_started`：项目 SQLite 维护任务开始。
10. `project:sqlite:maintenance_completed`：项目 SQLite 维护任务完成，包含成功状态、耗时和错误详情。

HTTP API 必须使用统一响应信封，字段使用 `snake_case`。

成功响应：

```json
{ "success": true, "data": {} }
```

失败响应：

```json
{ "success": false, "data": null, "error": { "code": "...", "message": "...", "details": "..." } }
```

## 6. 错误处理

必须覆盖：

1. 未选择项目。
2. 正在打开项目。
3. `~/.avcs/avcs.sqlite3` 初始化失败。
4. 项目 SQLite 初始化失败或迁移失败。
5. 项目文件夹已移动或全局 SQLite 映射失效。
6. 空白项目名称为空或包含不允许的路径字符。
7. Phoenix 文件 API 路径校验失败或文件操作失败。

## 7. 验收标准

1. 用户可以通过 Phoenix 本地 API 新建空白项目或打开一个本地文件夹作为项目。
2. 新建空白项目会在 `projects.default_root` 指定目录下创建不重名的项目目录，默认回退 `~/Documents/Avcs`。
3. 打开项目时会创建或复用 `.avcs/project.sqlite3`、`work/` 和 `output/`。
4. 再次打开同一文件夹后可以恢复项目数据。
5. 全局 SQLite 不写入项目业务数据。
6. 项目路径失效时提示用户重新打开文件夹，不自动修复 SQLite 映射。
7. React 不使用 Chromium File System Access API。
8. 用户可以从项目行菜单查看当前项目 SQLite 状态，并显式触发快速优化或深度整理。
9. 全局项目索引为空时，打开工作台会自动弹出一次新建空白项目 Dialog；取消后不重复打扰。
