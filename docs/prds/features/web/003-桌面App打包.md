# 桌面 App 打包

状态：Draft
日期：2026-06-02

## 1. 背景

Avcs MVP 的主要形态仍是 Phoenix 承载的本地 Web 应用。为了便于本机启动和后续交付验证，Avcs 可以被打包为 macOS Apple Silicon 桌面 app。

桌面 app 不引入新的业务运行时。Tauri 负责启动打包进 app bundle 的 Phoenix release，维护托盘菜单、deep link、文件关联和 updater 入口，并通过系统默认浏览器打开 `http://127.0.0.1:<port>/web/`。

## 2. 目标

1. 只支持 macOS Apple Silicon 构建，目标为 `aarch64-apple-darwin`。
2. 将 Phoenix release 作为 Tauri app resource 打包进 `.app`。
3. 桌面启动时动态选择本地端口，Phoenix 只绑定 `127.0.0.1`。
4. 关闭 Tauri app 时终止 Phoenix release 子进程。
5. 支持 `avcs://` deep link 和 `.avcs` 文件关联，把路径转为 `/web/?project_path=...` 后由浏览器前端调用 Phoenix API 打开项目。
6. 启用 macOS arm64 Tauri updater，使用 GitHub latest `latest.json`、Tauri signing key 和托盘菜单检查更新。
7. CI 上传 updater artifacts 和手动 `.app.zip` artifact，仍不做 notarization。

## 3. 边界

1. React 仍然只通过 Phoenix HTTP API 和 WebSocket 与后端通信。
2. React 不直接调用 Tauri IPC，不直接访问 SQLite、Codex app-server 或本地文件系统。
3. 文件导入、打开目录、复制路径等能力仍通过 Phoenix API 实现。
4. Codex app-server 仍由 Avcs 后端封装启动。
5. 不实现 Windows、Linux、Intel macOS、DMG、Developer ID/P12正式签名或 Apple notarization。

## 4. 运行方式

Tauri 启动后：

1. 绑定 `127.0.0.1:0` 选择可用端口，并建立 Tauri 与 Phoenix release 的 ElixirKit PubSub 通道。
2. 启动 app resource 中的 Phoenix release。
3. 注入 `AVCS_DESKTOP=true`、`PHX_SERVER=true`、`PORT=<selected_port>` 和 `ELIXIRKIT_PUBSUB=<pubsub_url>`。
4. Phoenix endpoint 启动后，桌面桥接进程通过 PubSub 广播 ready URL。
5. Tauri 收到 ready 后启用托盘菜单，并通过系统默认浏览器打开 `/web/`。
6. Tauri 把托盘菜单、deep link 和文件关联事件转发给 Phoenix 侧处理。

桌面 release 使用 `~/.avcs/runtime.sqlite3` 作为 Phoenix `Avcs.Repo` 的运行时占位库。项目业务数据仍保存在 `<project>/.avcs/project.sqlite3`。

## 5. 验收标准

1. `rel/app/tauri.sh build --target aarch64-apple-darwin` 能构建 `.app`。
2. `rel/app/tauri.sh app --target aarch64-apple-darwin` 能启动 Avcs 并在系统默认浏览器打开 Web 应用。
3. `/api/health` 返回 `{ "success": true, "data": { "status": "ok" } }`。
4. 关闭 app 后没有残留 Phoenix release 子进程。
5. 托盘菜单能重新打开 Avcs、打开设置页、复制本地 URL、打开日志，并能手动检查更新。
6. `avcs://settings` 能打开设置页；`avcs://open?path=/absolute/project/folder` 能打开对应项目。
7. GitHub Actions 上传 `latest.json`、macOS arm64 `.app.tar.gz`、`.sig` 和 `Avcs-macos-aarch64.app.zip`。

## 6. 已知限制

CI 通过 Tauri build 流程生成 updater artifacts，并用 `TAURI_SIGNING_PRIVATE_KEY` / `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` 签名更新包。当前仍未做 Apple notarization；macOS 对下载后的 artifact 仍可能提示 unidentified developer（属于未公证的预期行为）。如果 Tauri updater 私钥丢失或被覆盖，已安装版本无法继续验证后续用新 key 签名的更新。
