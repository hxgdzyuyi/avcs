# 012 三栏工作台 UI

状态：Done  
领域：`web/src`

## 1. 目标

Avcs MVP 默认使用三栏工作台，让界面从「功能可用的三栏界面」升级为「低干扰、画板优先、可长时间工作的创作工作台」。

本功能只细化 UI 表现、信息层级和交互状态，不改变以下边界：

1. Avcs 是 Phoenix 承载的本地 Web 应用。
2. 前端使用 React + Vite + 纯 JavaScript。
3. 样式使用 Sass/SCSS，不使用 Tailwind、CSS-in-JS、Less 或组件库主题系统。
4. 聊天输入使用 CodeMirror。
5. 本地文件操作全部通过 Phoenix API。
6. Agent 调用全部通过 Codex app-server。
7. 画板使用普通 HTML DOM 元素自由布局，不使用 Canvas/WebGL/SVG 作为主渲染层。

## 2. 设计原则

1. 低对比工作台：整体使用白色、浅灰、细边框和少量高亮色，避免强背景和重阴影。
2. 固定侧栏 + 任务列 + 大画布：左侧承载项目导航，中间承载对话与生成记录，右侧把绝大部分面积留给视觉结果。
3. 顶部轻量模式切换：用分段控件和图标按钮承载低频操作，不抢占主要内容。
4. 对话列窄而完整：聊天区宽度固定，消息、结果链接和底部输入器形成完整闭环。
5. 画布区域留白充足：视觉对象分组陈列在大面积留白中，用户能快速比较多个版本。
6. 悬浮工具就近出现：画板工具、筛选和缩放控件贴近画布边缘，降低顶部工具栏复杂度。
7. 主要操作用图标表达：上传、扫描、引用、复制路径、打开文件夹、定位等操作优先使用 lucide 图标按钮，并提供 tooltip 或 `aria-label`。

## 3. 目标布局

```text
┌──────────────┬─────────────────────────────── top bar ───────────────────────────────┐
│ Project pane │ Chat / thread pane │                  Board workspace                 │
│ 228px        │ 456px              │                  fluid                           │
└──────────────┴────────────────────┴──────────────────────────────────────────────────┘
```

桌面尺寸：

1. 左栏 `ProjectPane`：固定宽度 `228px`，最小 `220px`，不随窗口拉伸。
2. 中栏 `ChatPane`：固定宽度 `456px`，最小 `380px`，最大 `520px`。
3. 右栏 `BoardPane`：占据剩余空间，最小可用宽度 `640px`。
4. 顶栏高度 `60px`，可横跨聊天区和画板区；左栏保留自己的品牌和项目入口。
5. 应用整体高度为 `100vh`，三个主区域内部独立滚动。

窄屏降级：

1. `<= 1080px`：隐藏右侧画板，保留项目栏和聊天栏；聊天中图片结果点击后可提示需要更宽窗口查看画板。
2. `<= 760px`：切换为单栏模式，用顶部 tab 在 `Project`、`Thread`、`Board` 三个视图间切换。
3. 移动端不是 MVP 的核心生产场景，但不能出现文字溢出、按钮重叠或无法发送消息。

## 4. 左栏 ProjectPane

左栏负责项目与 thread 导航，对齐 Codex 左侧栏的「项目管理器 + 会话列表」心智，不做完整文件浏览器。详细规格见 `docs/prds/features/ui/002-Codex左侧栏对齐.md`。

从上到下：

1. 顶部分类区：显示 `项目`，并提供侧栏展开/收起、更多菜单和文件夹加号入口。
2. 项目列表区：展示全局 SQLite 项目索引中的项目，支持切换项目、展开项目和显示项目状态。
3. Thread 列表区：展开项目下展示历史对话或任务 thread，支持选择、快捷键标记、最近活动时间和 `展开显示`。
4. 底部状态区：WebSocket / Agent / 文件 API 的简短状态；不要放无功能的营销卡片。

项目入口：

1. 文件夹加号菜单包含 `新建空白项目` 和 `使用现有文件夹`。
2. `新建空白项目` 要求用户输入项目名，由 Phoenix HTTP API 在 `~/Documents/Avcs` 下创建不重名的项目文件夹。
3. `使用现有文件夹` 要求用户输入本地绝对路径，并走 Phoenix HTTP API 打开或初始化该文件夹。
4. 顶部 `...` 菜单提供当前项目管理入口，至少包含归档当前项目和删除引用。
5. 删除引用只移除全局 SQLite 中的项目索引，不删除本地项目文件夹。
6. 未实现的项目操作不展示为可点击入口；需要保留入口时使用 disabled 状态。

视觉规则：

1. 背景使用 `#ffffff`，右边界使用 `1px` 浅灰线。
2. 当前项目和当前 thread 使用浅灰或浅蓝灰选中底，不使用大面积高饱和蓝色填充。
3. 项目行高 `32px` 到 `36px`，thread 行高 `30px` 到 `34px`，圆角不超过 `8px`。
4. 项目名、thread 标题和项目路径必须单行省略，hover 或 tooltip 展示完整内容。
5. 文件夹、加号、更多、展开收起等操作优先使用 `lucide-react` 图标，并提供 tooltip 或 `aria-label`。

空状态：

1. 未打开项目时，左栏仍可展示全局项目列表和项目添加入口。
2. 项目无 thread 时，在项目下展示 `暂无对话`，不展示空的 thread 列表噪声。
3. 禁用会触发写入的 thread 操作和聊天发送，直到项目上下文可用。

## 5. 中栏 ChatPane

聊天区是用户给 Agent 下达任务、引用图片和查看生成过程的地方。它应独立完整，但业务内容聚焦 thread / turn / item。详细规格见 `docs/prds/features/ui/003-Codex聊天区对齐.md`。

顶部 thread bar：

1. 左侧显示当前 thread 标题，超长省略。
2. 标题旁可显示 agent 状态：`idle`、`running`、`error`。
3. 右侧放上传图片、扫描项目图片、新建 thread 等图标按钮。
4. 不在顶部堆叠大量筛选控件；画板筛选属于 BoardPane。

消息列表：

1. 用户消息右侧气泡，浅灰背景，最大宽度约 `72%`。
2. Assistant 消息左侧正文，不强制气泡化，保持长文本可读。
3. 工具调用使用紧凑状态行，例如 `image_gen running`、`scan completed`。
4. 错误信息使用浅红背景或红色边线，必须说明失败来源。
5. 图片生成结果渲染为一组可点击的资产行。

底部 composer：

1. Composer 使用 CodeMirror，多行输入，最小高度 `148px`，最大高度 `280px`。
2. 引用图片缩略图显示在输入器上方或内部顶部，支持单个移除。
3. 左下角放添加图片按钮，支持上传或导入入口。
4. 右下角放发送按钮，使用明确的 action 色。
5. Agent 运行中发送按钮进入 loading / disabled 状态，但用户仍可编辑下一条草稿。
6. placeholder 保持一句话，例如 `Describe the image you want to create...`。

## 6. 右栏 BoardPane

画板是 Avcs 的主视觉区域，应比聊天区更安静、更宽、更适合比较多张图片。

画板结构：

1. `BoardPane` 背景使用极浅灰。
2. `board-canvas` 是白色或近白色自由工作区，默认不显示强网格。
3. 画板对象仍为 DOM 绝对定位元素。
4. 新生成或新导入资产默认放在当前视野中部，并按 turn 分组横向排列。
5. 分组标题使用小字号，不影响自由拖拽。

顶部与筛选：

1. 左侧显示当前画板范围：`All images`、`Current thread`、`Generated` 等。
2. 右侧使用一个筛选 pill 或 segmented control。
3. 筛选值使用业务语言，不直接暴露数据库字段。
4. 筛选改变只影响可见对象，不删除 board item，也不改变对象坐标。

画板对象：

1. 图片本体使用 `img`，`object-fit: contain`。
2. 未选中时只有极轻边框或无边框。
3. 选中时显示蓝色 `2px` 选中框、四角控制点、文件名和显示尺寸。
4. 控制点、名称、尺寸标注和操作按钮都用 DOM 元素实现。
5. 多选时显示 `n selected`，但不遮挡图片主体。
6. 拖动和缩放时即时更新前端位置，pointer up 后通过 WebSocket 保存。

悬浮工具：

1. `Select`：默认模式，点击选择，拖动移动对象。
2. `Reference`：将选中图片加入聊天输入引用。
3. `Reveal`：打开所在文件夹。
4. `Copy path`：复制本地路径。

MVP 若未实现 `Pan` 或缩放，不展示对应入口。

## 7. 顶栏与全局操作

MVP 最低要求：

1. 保留 `Canvas` 作为默认主视图。
2. `List` 视图如果未实现，不展示假入口。
3. Help / Search / Notifications 如果未实现，不展示无效按钮。
4. 中间保留空白，不塞入说明文案。

## 8. 视觉系统

推荐 token：

```scss
$font-sans: Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
$ink: #1f2328;
$muted: #737373;
$subtle: #a3a3a3;
$line: #dedede;
$soft-line: #ececec;
$panel: #f7f7f6;
$surface: #ffffff;
$workspace: #fbfbfa;
$selection: #2563eb;
$selection-weak: #eaf1ff;
$action: #ff6b2c;
$danger: #b42318;
$success: #15803d;
```

使用规则：

1. 大面积背景只使用白色和浅灰。
2. 选中框和画板对象操作使用蓝色。
3. 发送、生成等主动作可以使用橙色，面积要小。
4. 不使用大面积渐变、装饰光斑、深色英雄区或营销卡片。

字体与尺寸：

1. 基础字号 `14px`。
2. 左栏和工具条辅助文字 `12px`。
3. 顶部标题 `16px` 到 `18px`。
4. 行高保持 `1.4` 到 `1.55`。
5. 字间距保持 `0`，不使用负字距。

间距与圆角：

1. 基础间距使用 `4px`、`8px`、`12px`、`16px`、`24px`。
2. 面板边界用 `1px` 线，不使用重阴影。
3. 卡片和面板圆角不超过 `8px`。
4. pill、chip、segmented control 可以使用更大圆角，但只用于小型控件。
5. icon button 推荐 `32px`，紧凑操作可用 `28px`。

## 9. 前端实现映射

建议按以下文件边界推进：

1. `web/src/styles/_tokens.scss`：更新色彩、字体、间距、圆角和阴影 token。
2. `web/src/styles/_layout.scss`：调整三栏尺寸、顶栏、响应式行为和独立滚动。
3. `web/src/styles/_components.scss`：重写按钮、segmented control、消息、composer、画板对象和 notice。
4. `web/src/features/projects/ProjectPane.jsx`：调整项目区信息层级和 thread 行操作。
5. `web/src/features/chat/ChatPane.jsx`：增加 thread bar、资产结果行、composer 引用区域。
6. `web/src/features/board/BoardPane.jsx`：优化画板顶部筛选、对象默认分组、悬浮工具和选中态。

实现约束：

1. 不添加 TypeScript 文件或配置。
2. 不引入 Tailwind、CSS-in-JS、Less 或组件库主题系统。
3. 使用 `lucide-react` 里的现成图标。
4. 静态视觉规则写入 SCSS；画板对象坐标、尺寸和层级可继续由 React 写入 inline style 或 CSS custom properties。
5. React 不直接读取本地文件路径作为图片地址，图片预览继续走 Phoenix 受控 HTTP 路由。

## 10. 关键状态

UI 必须覆盖：

1. 未打开项目：左栏可打开项目，聊天和画板禁用或展示空态。
2. WebSocket 连接中、已连接、断开、重连失败。
3. Thread 为空：聊天区展示空消息列表和可用 composer。
4. Agent 运行中：聊天区显示流式文本、工具状态和不可重复发送状态。
5. 图片生成成功：资产行出现在聊天区，图片对象出现在画板中。
6. 图片生成失败：错误明确落在当前 turn 中，并保留用户输入。
7. 上传或导入失败：说明格式、大小、路径或权限问题。
8. 资产文件丢失：画板对象显示缺失占位，并提供重新扫描或打开文件夹入口。
9. 画板保存失败：拖动或缩放回滚到保存前位置，或显示可重试提示。

## 11. 验收标准

1. 桌面宽屏下，画板宽度明显大于聊天区，用户第一眼能看到视觉结果。
2. 左栏、聊天栏、画板区边界清晰，但没有重阴影或高对比装饰。
3. 当前 thread、当前项目、当前选中画板对象都有明确选中态。
4. 用户能从聊天中的图片结果一键定位到画板对象。
5. 用户能从画板对象一键加入聊天引用。
6. 画板对象拖动、缩放时不发生文字覆盖图片主体或按钮重叠。
7. 长项目路径、长 thread 标题、长文件名都能省略或换行，不撑破布局。
8. Agent 运行、工具调用、错误和空状态都有可见反馈。
9. 未实现的入口不展示为可点击按钮。
10. 页面在 `1440x900`、`1280x800`、`1080x720` 三档尺寸下没有主要内容重叠。

## 12. 不做事项

1. 不做营销 landing page。
2. 不做完整文件浏览器。
3. 不做专业图层树、文本对象、矢量编辑、蒙版、裁切、旋转或对齐吸附。
4. 不做 Canvas/WebGL/SVG 主画板。
5. 不展示无实际功能的 Help、Search、Notifications、List view、Pan 或 Zoom 入口。
6. 不引入新的设计系统依赖或组件库主题。
