# 013 Codex 左侧栏对齐

状态：Draft  
关联 PRD：`docs/prds/features/ui/001-三栏工作台UI.md`、`docs/prds/features/projects/001-项目打开与初始化.md`、`docs/prds/features/threads/001-Thread管理.md`

## 1. 目标

Avcs 左侧栏对齐 Codex 的项目与对话导航区。它的核心职责是让用户在多个本地项目、项目文件夹和历史任务线程之间切换，而不是提供传统 IDE 文件浏览器。

侧边栏需要支持：

1. 创建空白项目。
2. 使用已有本地文件夹作为项目。
3. 展示全局项目索引中的项目列表。
4. 展开当前或最近使用的项目，展示该项目下的 thread。
5. 切换项目与 thread，并驱动右侧聊天区、画板区同步上下文。
6. 展示 thread 快捷切换标记、最近活动时间和空状态。
7. 折叠过长 thread 列表，避免侧栏滚动负担过重。

## 2. 信息定位

左侧栏是「项目管理器 + 会话列表」，不是文件树。

用户心智：

1. 第一层是项目，对应 Avcs 全局 SQLite 中的项目记录和本地文件夹绑定关系。
2. 第二层是 thread，对应某个项目内的历史对话或任务。
3. 文件内容、命令输出、代码修改、Agent 执行过程和画板对象主要出现在右侧主工作区。
4. 图片资产列表和画板对象管理不进入左侧栏，避免侧栏变成混合资源管理器。

## 3. 框架布局

推荐宽度：

1. 默认宽度 `240px`。
2. 最小宽度 `220px`。
3. 最大宽度 `280px`。
4. 左栏内部独立滚动，不影响聊天区和画板区。

结构：

```text
┌────────────────────────┐
│ Header                 │
│ 项目       ↗  ...  +   │
├────────────────────────┤
│ Project list           │
│ ▾ avcs                 │
│   整理 docs/prds 文档  │
│   按002-ui改造界面  ⌘2 │
│   更新 002-ui 界面  ⌘3 │
│   展开显示             │
│ once-image2-mp         │
│ abyss-web              │
│ seedom-image           │
├────────────────────────┤
│ Status / connection    │
└────────────────────────┘
```

## 4. 顶部区域

顶部区域承载侧栏分类和项目入口。

元素从左到右：

1. `项目`：当前侧栏主分类。MVP 不做多分类切换时，保持静态文本。
2. 斜向箭头图标：用于展开、收起或调整侧栏显示状态。未实现折叠时不展示可点击态。
3. `...`：更多菜单，放置当前项目管理入口。MVP 至少包含 `归档当前项目` 和 `删除引用`；没有当前项目时禁用。
4. 文件夹加号按钮：创建或添加项目入口。

文件夹加号菜单：

1. `新建空白项目`：要求用户输入项目名，后端在全局软件设置 `projects.default_root` 下创建 `<projects.default_root>/<项目名>`，初始默认值为 `~/Documents/Avcs`。如果同名目录已存在，自动递增为 `<项目名> 2`、`<项目名> 3`。
2. `使用现有文件夹`：要求用户输入本地绝对路径作为项目，走 Phoenix HTTP API，不使用浏览器 File System Access API。

视觉规则：

1. 顶部高度 `48px` 到 `56px`。
2. 标题使用 `13px` 或 `14px` 中等字重。
3. 图标按钮使用 `28px` 或 `32px` 正方形点击区。
4. 菜单浮层宽度不小于 `184px`，菜单项高度 `36px` 到 `40px`。
5. 菜单项使用图标 + 文本，hover 只使用浅灰底。

## 5. 项目列表

项目行对应全局 SQLite 的项目记录。常见项目名称包括 `avcs`、`once-image2-mp`、`abyss-web`、`go-stable` 等。

项目行内容：

1. 展开指示：当前展开项目显示下箭头，未展开项目显示右箭头或省略指示。
2. 文件夹图标：表示项目或绑定的本地文件夹。
3. 项目名称：单行省略。
4. 状态补充：可选显示缺失、连接中、无对话等状态。
5. 右侧快捷操作：`…` 打开项目更多操作菜单，铅笔图标进入新会话准备状态。

交互：

1. 点击项目行会切换当前项目上下文。
2. 点击展开指示会展开或收起该项目下的 thread 列表。
3. 打开项目时，右侧聊天区和画板区同步到该项目的当前 thread 或默认 thread。
4. 点击铅笔图标时，右侧聊天区切换为空白输入界面，但不立即创建 thread。
5. 项目路径无效时，项目行保留，但显示失效状态，点击后提示重新选择文件夹。

项目排序：

1. 当前项目保持原列表位置，只通过选中态高亮，不因点击选择而置顶。
2. 项目显示顺序以全局 SQLite 中的 `projects.sidebar_order` 为准，按升序展示。
3. 创建新项目或首次 seed 旧数据时，最近活动时间、最近打开时间和名称只用于生成初始 `sidebar_order`；生成后不再直接参与列表排序。
4. 用户拖拽项目后，服务端重写完整 `sidebar_order`，后续刷新和推送都保持拖拽顺序。
5. 不单独做「最近项目」页面；左栏项目列表就是全局项目索引的主要入口。
6. `project:select` 只切换当前上下文，不计入最近活动或最近打开时间，也不改变 `sidebar_order`。

## 6. 展开的项目内容

展开项目后，项目行下方展示该项目的 thread。Thread 是历史对话或任务，不是文件。

Thread 行内容：

1. 标题：例如 `整理 docs/prds 文档`、`按002-ui改造界面`、`落地 PRD 功能`。
2. 快捷键标记：如 `⌘2`、`⌘3`、`⌘4`，只给最近或固定的少数 thread 显示。
3. 最近活动时间：如 `20 分`，用于表示最后更新时间。
4. 运行状态：Agent 运行中可显示小型状态点或 spinner。

Thread 排序：

1. Thread 显示顺序以项目 SQLite 中的 `threads.sidebar_order` 为准，按升序展示。
2. 创建新 thread 或首次 seed 旧数据时，创建时间倒序只用于生成初始 `sidebar_order`。
3. 用户拖拽 thread 后，服务端重写该项目下未归档 thread 的完整 `sidebar_order`，后续刷新和推送都保持拖拽顺序。
4. `created_at` 和 `updated_at` 继续用于展示时间和活动统计，不直接参与最终显示排序。

选中态：

1. 当前 thread 使用浅蓝色或浅灰蓝背景。
2. 选中态不能使用大面积高饱和蓝色。
3. 当前项目行与当前 thread 行都应可辨认，但 thread 选中态优先级更高。

行高与缩进：

1. 项目行高 `32px` 到 `36px`。
2. Thread 行高 `30px` 到 `34px`。
3. Thread 行相对项目行左缩进 `20px` 到 `28px`。
4. 行圆角不超过 `8px`。

## 7. 展开显示

当某个项目 thread 过多时，默认只显示前 `5` 到 `7` 个最相关 thread。

默认展示优先级：

1. 当前选中 thread。
2. 最近运行或最近打开 thread。
3. 固定快捷键 thread。
4. 最近更新时间倒序 thread。

`展开显示` 行：

1. 点击后展示更多 thread。
2. 再次点击可收起为默认数量，文案变为 `收起显示`。
3. 展开或收起只改变侧栏可见数量，不改变 thread 数据。
4. 如果剩余 thread 数量明确，可以显示 `展开显示 8 个`。

## 8. 空状态与项目状态

项目列表为空：

1. WebSocket join 后如果没有当前项目且全局项目索引为空，前端自动弹出一次 `新建空白项目` Dialog。
2. 自动 Dialog 复用文件夹加号菜单中的 `新建空白项目` 创建流程，不新增独立入口或后端 API。
3. 用户取消后保留左栏空状态，本次页面生命周期内不再次自动弹出；文件夹加号菜单仍可手动新建项目或使用现有文件夹。
4. 如果 URL 通过 `project_path` 指定要打开的外部项目，优先执行外部项目打开流程，不先弹出空项目 Dialog。

项目无 thread：

1. 在项目下展示 `暂无对话`。
2. `暂无对话` 不应像可点击 thread 一样高亮。
3. 聊天区可以保持可用 composer，用户发送第一条消息时创建 thread 和 turn。

项目文件夹失效：

1. 项目行显示失效状态，例如 `文件夹不可用`。
2. 点击项目后，在右侧或菜单中提供重新选择文件夹入口。
3. 不自动修复全局 SQLite 路径映射。

Agent 运行中：

1. 对应 thread 行显示轻量运行状态。
2. 切换到其他 thread 不应丢失当前运行 thread 的状态提示。

底部状态区：

1. 使用单行展示 WebSocket 与 Agent 状态，不展示文件 API 或 `Files` 状态。
2. 左侧显示 WebSocket 连接状态，例如 `online`、`connecting`、`offline`。
3. 右侧显示 Agent 状态；空闲为 `idle`，任意 thread 运行中显示 thinking 点阵。
4. 点阵由后端 `agent:thinking_tick` 推进，点满后从头开始，用于表达 Codex app-server 仍在产生交互活动。

连接断开：

1. 底部状态区显示 WebSocket 和 Agent 状态。
2. 项目和 thread 列表保持最近一次可用数据。
3. 禁用会触发写入的操作，并提示正在重连。

## 9. 操作菜单

顶部当前项目更多菜单包含：

1. 归档当前项目。
2. 删除引用。

项目行更多菜单建议包含：

1. 重命名项目。
2. 重新选择文件夹。
3. 打开所在文件夹。
4. 数据库情况：查看 `.avcs/project.sqlite3` 状态，并触发快速优化或深度整理。
5. 归档项目。
6. 删除引用。

Thread 更多菜单建议包含：

1. 重命名 thread。
2. 固定到快捷键。
3. 复制 thread 标题。
4. 归档 thread。
5. 删除 thread。

MVP 规则：

1. 未实现的菜单项不要展示为可点击按钮。
2. 删除操作必须确认。
3. 归档优先于硬删除；删除引用只移除全局 SQLite 中的项目索引，不删除本地项目文件夹。
4. 菜单操作仍走 Phoenix API 或 WebSocket，不从 React 直接读写 SQLite。
5. 数据库情况入口只展示当前项目 SQLite 信息；深度整理必须由用户显式点击触发。

## 10. 快捷键标记

快捷键标记用于快速切换最近或固定的 thread。

规则：

1. 只显示 `⌘1` 到 `⌘9` 范围内的少数条目。
2. `⌘1` 可以保留给当前主要 thread 或最常用入口；具体映射需在实现中统一。
3. 标记只作为提示，不参与标题文本复制。
4. 快捷键冲突时以当前项目内的可见 thread 为准。

视觉：

1. 使用 `11px` 辅助字号。
2. 颜色使用 muted token。
3. 不使用边框 chip，避免侧栏变重。

## 11. 数据与事件映射

需要的数据：

1. Project：`id`、`name`、`folder_path`、`last_opened_at`、`last_activity_at`、`status`。
2. Thread：`id`、`project_id`、`title`、`updated_at`、`status`、`is_archived`、`shortcut_index`。
3. Sidebar UI state：`expanded_project_ids`、`selected_project_id`、`selected_thread_id`、`show_all_thread_project_ids`。

推荐事件：

1. `projects:list`：读取全局项目列表。
2. `project:create_blank`：根据用户输入的项目名，在全局软件设置 `projects.default_root` 下创建空白项目。
3. `project:open_folder`：根据用户输入的本地绝对路径，使用已有文件夹打开或创建项目。
4. `project:select`：切换项目。
5. `project:archive`：归档项目，默认项目列表不再显示，不删除文件夹。
6. `project:delete`：删除项目引用，只移除全局 SQLite 记录，不删除文件夹。
7. `threads:list`：读取某项目下 thread。
8. `thread:select`：切换 thread。
9. `thread:create`：在当前项目的新会话准备态发送第一条消息时创建 thread；点击铅笔入口不创建 thread。
10. `sidebar:state:update`：保存展开、收起等 UI 状态；也可以先只保存在前端内存。

服务端推送：

1. `projects:updated`：项目列表变化。
2. `threads:updated`：thread 列表变化。
3. `thread:updated`：单个 thread 标题、状态或时间变化。
4. `agent:run_started`、`agent:run_completed`：驱动 thread 运行状态。
5. `error`：项目、thread 或连接错误。

## 12. 视觉规范

颜色：

1. 背景使用 `#ffffff` 或 `#f7f7f6`。
2. 右边界使用 `#dedede` 或 `#ececec`。
3. hover 使用浅灰底。
4. 当前 thread 选中态使用浅蓝灰，例如 `#eaf1ff`。
5. 危险状态只在图标或文字上使用红色，不铺满整行。

文本：

1. 项目标题 `13px` 到 `14px`。
2. Thread 标题 `13px`。
3. 状态、时间和快捷键 `11px` 到 `12px`。
4. 项目名、thread 标题和路径都必须支持单行省略。

图标：

1. 文件夹、加号、更多、展开收起、斜向箭头优先使用 `lucide-react` 图标。
2. 不手写 SVG 图标，除非 lucide 没有对应图标。
3. 图标按钮必须有 `aria-label` 或 tooltip。

## 13. 响应式与折叠

桌面：

1. 左栏常驻。
2. 允许收起为窄栏时，只保留项目图标、当前项目状态和展开按钮。
3. 收起状态下 hover 或点击可临时展开。

窄屏：

1. `<= 1080px` 仍保留项目与 thread 导航，但可隐藏右侧画板。
2. `<= 760px` 使用单栏 tab 时，项目导航作为一个独立视图。
3. 菜单和 thread 标题不得溢出屏幕。

## 14. 验收标准

1. 用户能在左栏看到全局项目索引中的多个项目。
2. 用户能通过文件夹加号菜单新建空白项目或使用现有文件夹。
3. 当前项目可展开并显示该项目下的 thread。
4. 当前 thread 有明确选中态，并驱动右侧主区显示对应对话。
5. 项目下 thread 过多时能通过 `展开显示` 控制可见数量。
6. 空项目显示 `暂无对话`，不展示空白列表噪声。
7. 最近活动时间、运行状态和快捷键标记不会挤压标题导致布局破裂。
8. React 不直接访问本地文件、SQLite 或 Codex app-server。
9. 侧栏没有文件树、资产网格、营销卡片或无功能入口。
